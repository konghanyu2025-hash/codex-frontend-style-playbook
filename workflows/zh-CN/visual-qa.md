# 视觉 QA 工作流

任何使用本手册构建的前端在交付前都应执行这个工作流。

## 必查视口

至少检查：

- 桌面端：1440 x 900。
- 笔记本：1280 x 800。
- 移动端：390 x 844。

如果产品有明确设备目标，应增加对应视口。

## 截图审查清单

检查截图中是否存在：

- 文本溢出。
- 按钮文字裁切。
- 卡片或面板重叠。
- 导航折叠问题。
- 首屏空洞。
- 层级弱。
- 色彩单调。
- 装饰元素过多。
- 间距不一致。
- 缺少聚焦、悬停、选中、空、加载或错误状态。
- 重要内容无理由地落在首屏之外。

## 浏览器检查

对于需要开发服务器的项目：

1. 启动应用。
2. 打开桌面端视口。
3. 截图。
4. 打开移动端视口。
5. 截图。
6. 与主要控件交互。
7. 修复可见问题。
8. 重复直到截图匹配 brief。

## 自动检查

如果可用，使用 Playwright 或项目现有浏览器测试工具。

推荐断言：

- 移动端没有横向滚动。
- 主导航可见，或按设计折叠。
- 主操作可见。
- 没有明显控制台错误。
- 关键控件可键盘聚焦。
- 主内容不是空白。

## Visual QA 输出

创建或更新 `visual_acceptance_checklist.md`：

```md
# Visual Acceptance Checklist

## Screenshots Reviewed

## Issues Found

## Fixes Made

## Remaining Risks

## Acceptance Decision
```

## 验收标准

如果 UI 存在明显文本重叠、控件裁切、移动端破损、主区域空白或与 implementation brief 明显漂移，不要交付。
