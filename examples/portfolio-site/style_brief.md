# Style Brief

## Product Goal

Create a portfolio site that helps an independent interaction designer present credibility, selected work, and a clear path to contact.

## Target User

Product leaders, founders, and design directors evaluating whether the designer can handle complex interface work.

## Required Screens or Sections

- First viewport with name, role, and selected proof.
- Featured case studies.
- Process snapshot.
- Client or domain credibility.
- Contact section.

## Information Density

Medium. The site should feel editorial and distinct, but work examples must be inspectable without excessive scrolling.

## References

- Independent designer portfolios with strong typography.
- Editorial product case study pages.
- Data visualization portfolios with structured project previews.

Do not copy individual layouts, names, case study content, or image treatments.

## Visual Direction

Confident, editorial, and product-aware. The first viewport should immediately signal the person and the work type. Use strong typography, structured project previews, and one memorable visual system.

## Constraints

- Must keep the name and role visible in the first viewport.
- Must show a hint of work before the fold.
- Must avoid vague creative fluff.
- Must remain readable on mobile.

## Anti Goals

- No generic split hero with a card on one side.
- No abstract gradient background as the main visual idea.
- No portfolio that hides work behind decorative motion.
- No tiny nav-only brand signal.
