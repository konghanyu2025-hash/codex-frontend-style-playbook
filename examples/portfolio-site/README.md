# Example: Portfolio Site

This example shows how the playbook should guide an expressive but usable portfolio.

The task:

```text
Build a portfolio site for an independent interaction designer who creates data-heavy product interfaces.
```

Files:

- `style_brief.md`: product and style direction.
- `visual_anchor.md`: generated-anchor review notes.
- `implementation_brief.md`: concrete frontend rules.
- `visual_acceptance_checklist.md`: QA checklist.

Use this example as a workflow reference, not a template to copy.
